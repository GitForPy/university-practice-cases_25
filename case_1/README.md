# Кейс-задача №1: Алгоритм обработки массива

## Описание задачи

Разработать алгоритм для вычисления суммы отрицательных элементов массива, расположенных между минимальным и максимальным элементами (границы не включаются).

## Постановка задачи

**Входные данные:** массив целых чисел произвольной длины.

**Выходные данные:** сумма всех отрицательных элементов, находящихся между позициями минимального и максимального элементов массива.

**Ограничения:**
- Минимальная длина массива: 3 элемента
- Границы (сами минимальный и максимальный элементы) не учитываются
- Если массив содержит менее 3 элементов, результат равен 0

## Алгоритм решения

1. Проверка длины массива (минимум 3 элемента)
2. Поиск минимального элемента и его индекса
3. Поиск максимального элемента и его индекса
4. Определение границ диапазона для суммирования
5. Суммирование отрицательных элементов в диапазоне

## Сложность алгоритма

- Временная сложность: O(n), где n - количество элементов массива
- Пространственная сложность: O(1)

## Тестовые случаи

| Массив | Результат | Пояснение |
|--------|-----------|-----------|
| [3, -1, 4, -2, 5, -3, 1] | 0 | min=-3(idx=5), max=5(idx=4), элементов между ними нет |
| [1, -2, 3, -4, 5, -1, 2] | 0 | min=-4(idx=3), max=5(idx=4), элементов между ними нет |
| [10, -5, 3, -2, 1, 8, -4] | -6 | min=-5(idx=1), max=10(idx=0), между ними: -2, -4 |
| [-1, 5, -3, 2, -4, 8, 1] | 0 | min=-4(idx=4), max=8(idx=5), элементов между ними нет |
| [2, -3, 5, -1, -2, 7, 1] | -3 | min=-3(idx=1), max=7(idx=5), между ними: -1, -2 |
