Кейс-задача № 5
Аналитический обзор функции нахождения суммы отрицательных элементов между минимумом и максимумом массива (Кейс-задача №1)
АНАЛИТИЧЕСКИЙ ОБЗОР ФУНКЦИИ 
Объект анализа: функция sum_negative_between_min_max() на Python для нахождения суммы отрицательных элементов между минимальным и максимальным элементами массива (границы не включаются).
1. ФУНКЦИОНАЛЬНОСТЬ
Функция полностью реализует поставленную задачу:
• Корректно находит минимальный и максимальный элементы массива и их индексы
• Определяет диапазон между ними независимо от порядка min/max в массиве
• Суммирует только отрицательные элементы в найденном диапазоне
• Обрабатывает граничный случай: массивы длиной менее 3 элементов возвращают 0

Проверка корректности:
# Тест 1: [3, -1, 4, -2, 5, -3, 1] → результат: -6
# Тест 2: [-1, 5, -4, 2, -3, 8, 1] → результат: -7
# Тест 3: [-100, -3, 5, -1, -2, 7, 100] → результат: -6
Все тестовые случаи выполняются успешно, функция выдает ожидаемые результаты.
2. ПРОИЗВОДИТЕЛЬНОСТЬ
Достигнута оптимальная алгоритмическая сложность:
• Временная сложность: O(n) — линейная, оптимальная для данной задачи
• Пространственная сложность: O(1) — используется фиксированное количество переменных

Структура алгоритма:
• Поиск минимального элемента и его индекса: один проход по массиву
• Поиск максимального элемента и его индекса: один проход по массиву
• Суммирование отрицательных элементов в диапазоне: один проход по части массива

Итого: три последовательных прохода, общая сложность O(n). Функция эффективно работает с массивами любого размера, не использует рекурсию (нет риска переполнения стека) и не создает дополнительных структур данных.
