АНАЛИЗ РЫНКА ИНФОРМАЦИОННЫХ СИСТЕМ С ВЕБ-АРХИТЕКТУРОЙ

На российском рынке программного обеспечения представлены различные информационные системы, построенные по архитектуре веб-приложений. В качестве объекта анализа выбрана система электронного документооборота Directum RX (разработчик: ООО "Directum", Россия).
Directum RX представляет собой цифровую платформу для управления бизнес-процессами и документооборотом предприятий с полноценным веб-интерфейсом. Система используется более чем в 5000 организациях различных отраслей.

АРХИТЕКТУРА СИСТЕМЫ DIRECTUM RX

Тип архитектуры
Directum RX реализована на основе трехзвенной архитектуры с элементами микросервисного подхода:

• Уровень 1. Представления (Presentation Layer): веб-клиент на базе HTML5, CSS3, JavaScript; десктоп-клиент (Windows); мобильные приложения (iOS, Android).

• Уровень 2. Бизнес-логики (Business Logic Layer): Web Agent (маршрутизация на базе IIS); Application Server (обработка бизнес-процессов); Background Services (фоновые задачи); Integration Services (интеграция с внешними системами); Conversion Services (преобразование документов).

• Уровень 3. Данных (Data Layer): СУБД Microsoft SQL Server, PostgreSQL, Postgres Pro; распределенное файловое хранилище; индексная база полнотекстового поиска.

Технологический стек
• Backend: .NET Framework / .NET Core
• Frontend: HTML5, CSS3, JavaScript (SPA)
• API: REST API для интеграций
• СУБД: MS SQL Server 2016+, PostgreSQL
• Web-сервер: Microsoft IIS
• Протоколы: HTTPS, REST, SOAP

ОЦЕНКА ВОЗМОЖНОСТЕЙ ВЕБ-АРХИТЕКТУРЫ

Преимущества архитектурного решения
Масштабируемость. Горизонтальное масштабирование до 50,000 одновременных пользователей. Хранение сотен миллионов документов без снижения производительности.

Доступность. Кроссплатформенный доступ через веб-браузер. Работа из любого местоположения. Не требуется установка клиентского ПО.
Производительность. Время отклика системы составляет доли секунды. Оптимизированные SQL-запросы и кэширование.
Безопасность. Многоуровневая аутентификация и авторизация. Шифрование данных по протоколу HTTPS. Сертифицированные средства электронной подписи (ФСБ России).
Отказоустойчивость. Микросервисная архитектура предотвращает единую точку отказа. Кластеризация и репликация данных.
Ограничения веб-архитектуры
• Зависимость от стабильности интернет-соединения
• Функциональные ограничения браузерной среды выполнения
• Требования к пропускной способности канала связи
• Необходимость поддержки актуальных версий веб-браузеров

ВАРИАНТЫ ИСПОЛЬЗОВАНИЯ В КОМПАНИИ

По размеру организации:

Малые предприятия (10-100 сотрудников): облачная модель развертывания (SaaS); базовый документооборот и электронный архив; интеграция с бухгалтерскими системами.

Средние предприятия (100-1000 сотрудников): on-premise развертывание на собственных серверах; управление производственной документацией; автоматизация согласований и утверждений.

Крупные корпорации (1000+ сотрудников): гибридная архитектура; интеграция с корпоративными информационными системами (ERP, CRM); территориально распределенная работа.

По отраслям применения

Государственный сектор. Электронное делопроизводство по ГОСТ, межведомственное взаимодействие, работа с обращениями граждан.

Энергетика и ЖКХ. Обработка заявок потребителей, управление договорами энергоснабжения, техническая документация, регуляторная отчетность.

Финансовый сектор. Управление кредитными досье, согласование финансовых операций, комплаенс и внутренний контроль.
Торговля и дистрибуция. Согласование договоров с поставщиками, управление закупками, документооборот розничной сети.
